형상관리(SCM: Software Configuration Management)

1. 변경 관리: 소프트웨어의 변경(수정 사항 - 데이터 베이스, 코드, 테스트)

2. 버전 관리
- 풀 모델 체인지: 외형뿐만이 아닌 엔진까지도 바뀌는 경우 ex) Ver.1.0 -> Ver.2.0
- 마이너 체인지: 외형 정도만 바뀌는 경우. ex) Ver.1.0 -> Ver.1.1.1	V(주번호)(부번호)
릴리스 버전(release version): 현재 사용 가능한 버전


3. 형상 관리: 소프트웨어 개발을 위한 전체 과정에서 발생하는 모든 항목의 변경사항을
관리하기 위한 활동. 적절한 변경관리를 통해 무절제한 변경을 사전에 예방하고 변경에
따른 부작용을 최소화함

형상관리 도구
- 공유 폴더 방식: 버전 관리 자료가 로컬 컴퓨터의 공유 폴더에 저장되어 관리 RCS
- 클라이언트/서버 방식: 버전 관리 자료가 서버에 저장되어 관리 CVS, SVN
- 분산 저장소 방식: 버전 관리 자료가 하나의 원격 저장소와 분산된 개발자의 PC의
로컬 저장소에 함께 저장되어 관리되는 방식
	Bitkeeper: 대규모 프로젝트에서 빠른 속도를 내도록 개발됨
	git: git의 속도에 중점을 둔 분산형 버전 관리 시스템
- Subversion: 중앙 서버에서 모든 소스코드를 보관 관리
